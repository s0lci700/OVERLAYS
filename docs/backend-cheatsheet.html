When to Call What
You want to… Call
Mutate HP characterActions.updateHp(id, hp)

Remove condition by ID characterActions.removeConditionById(id, condId)

In-Memory State (no PocketBase)
state/encounter.ts
Active combatants, turn index, round counter. Reset on endEncounter.
state/scene.ts
Current scene name + focused character. No persistence across server restart.
socket/rooms.ts
Holds the _io Server ref. broadcast() reads it at call-time.

PocketBase singleton + auth. ensureAuth() re-authenticates if token expired.
Rules
✅ Use characterActions for any mutation that needs to broadcast. It guarantees PocketBase write → broadcast atomicity.
✅ Use domain/* functions for business logic before writing to PocketBase — they are pure and testable without mocking.
⚠️ pb.collection().getOne() throws ClientResponseError on 404 — always wrap in try/catch, not an if (!result) guard.
⚠️ Never import data/* into frontend files — it pulls in Node.js crypto and breaks the browser bundle.
🚫 Never call io.emit() directly — always go through broadcast() so events are JSONL-logged to logs/sidecar.jsonl.
