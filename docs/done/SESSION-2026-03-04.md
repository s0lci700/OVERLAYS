### 5. PocketBase 404 Fix

**Root cause:** `pb.collection().getOne()` throws `ClientResponseError` (with `.status === 404`) when a record is not found â€” it does **not** return `null`. All `if (!character)` guards in `server.js` were dead code.

**Fix applied in `data/characters.js`:**
```js
async function findById(pb, id) {
  try {
    return await pb.collection("characters").getOne(id);
  } catch (err) {
    if (err?.status === 404) return null;
    throw err;
  }
}
```

Also added a null guard in `updateHp()` before accessing `character.hp_max`.

**Fix applied in `server.js`** â€” all affected route handlers now use try/catch:
- `PUT /api/characters/:id/hp`
- `PUT /api/characters/:id/photo`
- `PUT /api/characters/:id`
- `POST /api/characters/:id/conditions`
- `DELETE /api/characters/:id/conditions/:condId`
- `DELETE /api/characters/:id`
- `PUT /api/characters/:id/resources/:rid`
- `POST /api/characters/:id/rest`

### 7. JSONL Sidecar Logger

Every `io.emit()` in `server.js` is now intercepted by a `broadcast()` wrapper:

```js
function broadcast(event, data) {
  io.emit(event, data);
  logEvent(event, { /* simplified summary â€” no full character arrays */ });
}
```

`logEvent()` writes one line to `logs/sidecar.jsonl`:
```json
{ "event": "hp_updated", "ts_abs": 1709518800000, "ts_rel": 42300, "charId": "CH101", "charName": "Kuminak" }
```

- `ts_abs` â€” wall-clock Unix ms, always present
- `ts_rel` â€” ms since `SYNC_START` was fired; `null` until then
- Character arrays are never written to the log (only `charId`/`charName` summaries)
- `logs/` dir is created on server startup if it doesn't exist

### 9. Transaction Manager (Batch HP)

New endpoint: `POST /api/characters/batch/hp`

Body: `{ updates: [{ charId, hp_current }] }`

- Runs all PocketBase writes in parallel (`Promise.all`)
- Each success emits its own `hp_updated` broadcast (overlays update one by one)
- Returns `{ results: [...], errors: [...] }` â€” partial success allowed
- Use case: AOE damage, long-rest HP refill for the whole party
