D&D 5e Rules Foundation

How Combat Works in D&D 5e

The system implements the D&D 5th Edition combat loop (PHB p.189). Every mechanical decision in the codebase maps to one of these phases.

Phase 01

Surprise Check

DM determines if any combatants are surprised (no actions or reactions on turn 1). Handled narratively — not yet tracked in the system.

Phase 02

Roll Initiative — d20 + DEX Modifier

Every combatant rolls d20 + their DEX modifier. The DM enters rolls manually in the Initiative Strip. Ties broken automatically by DEX mod in the app.

Phase 03 → Repeated

Combat Rounds (6 seconds each)

Each combatant takes a turn in initiative order. A full cycle = 1 Round. The app tracks round number and advances the active turn pointer on "Siguiente ▶".

Each Turn

Move + Action + Bonus Action + Reaction

Movement (up to Speed ft.), one Action, one Bonus Action (if available), and one Reaction (triggered off-turn). The DM resolves these narratively; the app tracks HP and condition outcomes.

Outcomes

HP Changes, Conditions, Death Saves

Damage reduces HP. Healing restores it. Conditions (Poisoned, Frightened, etc.) alter capabilities. At 0 HP, players make Death Saving Throws (3 successes = stable, 3 failures = dead). Death saves not yet tracked in the system.

End State

Combat Ends → Short or Long Rest

Combat ends when all enemies are defeated (or fled). Party recovers with a Short Rest (hit dice, some abilities) or Long Rest (full HP, all resources). The app handles both via the Rest button.

DM Session Panel · /session

DM Actions During Combat

From the Session Panel, the DM applies outcomes in 2 taps: select action → select target → confirm. Updates broadcast instantly via Socket.io.

⚔

Damage

ACTION · tap + amount

Reduces target HP. Clamped to 0 (can't go negative). Triggers critical animation on overlay if HP ≤ 30%.

✦

Heal

ACTION · tap + amount

Restores HP. Clamped to hp_max. Triggers heal pulse on HP bar.

◈

Condition

ACTION · tap + name

Adds a named condition badge (Poisoned, Frightened, etc.) to the character. Visible on overlay and DM panel immediately.

☽

Rest

ACTION · tap + short/long

Short rest: recovers hit dice HP. Long rest: full HP + all resource pools restored.

▶

Next Turn

ALWAYS AVAILABLE

Advances initiative pointer. Wraps at end → increments round counter. Active token in strip gets cyan glow.

Active Conditions Reference

Poisoned

Disadvantage on attack rolls and ability checks. Hector is currently Poisoned — Giant Rat bite, DC 10 Con save.

Frightened

Disadvantage on ability checks and attacks while source is visible. Can't willingly move closer to source. Luis and Wort are Frightened.

Prone

Attacks against have advantage (melee) or disadvantage (ranged). Own attacks have disadvantage. Half speed to stand.

Blinded

Attacks against have advantage. Own attacks have disadvantage. Fails sight-based ability checks.

Restrained

Speed 0. Attacks against have advantage. Own attacks and Dex saves have disadvantage.

Stunned

Incapacitated (no actions/reactions). Fails Str and Dex saves automatically. Attacks against have advantage.

Honest Assessment · Future Work

What's Not Yet Tracked

Death Saving Throws

At 0 HP, D&D requires 3 successes/failures on d20 rolls. Currently the app only shows 0 HP — death save tracking UI is planned for next sprint.

Enemy HP (Live)

npc_instances have hp_current seeded, but damage to enemies isn't wired through the DM panel yet. The DM tracks enemy HP mentally or on paper.

Surprise Round

Surprised creatures can't act on their first turn. Purely narrative right now — no mechanical enforcement in the initiative strip.

Concentration Tracking

Cynthia's Hex requires Concentration. Taking damage requires a DC 10 Con save or the spell drops. Not automatically tracked.
