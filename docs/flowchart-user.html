Reference

Glossary

Key terms you'll see when using or discussing this system.

OBS Studio

Open Broadcaster Software â€” the free program used to record and stream. The overlays appear as transparent layers on top of the game or camera feed.

Browser Source

A type of source in OBS that loads a webpage. We use it to display the HP bars and dice results as live, animated graphics on top of the stream.

Socket.io

The technology that keeps the phone, server, and overlays connected in real-time. Think of it as a group chat that instantly delivers messages to all participants the moment something changes.

Control Panel

The mobile-friendly website the game master uses to control everything. Accessible on any phone or tablet connected to the same Wi-Fi as the laptop.

HP (Hit Points)

A character's health in D&D. When it reaches 0, the character goes unconscious. The system tracks current HP and maximum HP for every character.

Natural 20 / Natural 1

On a d20 (20-sided die), rolling exactly 20 is a "critical success" (Â¡CRÃTICO!) and rolling exactly 1 is a "critical failure" (Â¡PIFIA!). The overlay shows special effects for these.

REST (Short / Long)

In D&D, characters can recover abilities by taking a short rest (1 hour) or long rest (8 hours). The system automatically restores the correct resources for each type of rest.

Conditions

Status effects in D&D like Poisoned, Stunned, or Invisible. The game master can add or remove these per character, and they're tracked in real time.
