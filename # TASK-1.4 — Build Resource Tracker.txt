# TASK-1.4 — Build Resource Tracker Component

## Task ID

`TASK-1.4`

## Title

Build ResourceTracker component for character resources (hit dice, spell slots, class features)

## User Story

As a **player**, I want to **track my character's limited resources (hit dice, spell slots, class features like Second Wind) and see current vs. maximum values** so that **I know how many uses I have left and can plan my actions during combat**.

## Background / Context

Character resources are tracked in the `resources` JSON field of the `characters` PocketBase record. This field contains all character-specific limited-use abilities:

**Example `resources` JSON:**

```json
{
  "hit_dice": { "current": 4, "max": 4, "die": "d10" },
  "second_wind": { "current": 1, "max": 1 },
  "action_surge": { "current": 1, "max": 1 },
  "spell_slots": {
    "level_1": { "current": 4, "max": 4 },
    "level_2": { "current": 3, "max": 3 }
  }
}
```

The ResourceTracker component displays these resources in a visual, easy-to-scan format. Depending on permissions:

- **Players** see their resources but cannot modify (read-only display)
- **DMs** and **Stage operators** can increment/decrement (future: socket-driven mutations)

This task focuses on the **read-only player view**. Increment/decrement buttons may be added in TASK-1.6 or later when socket-driven mutations are implemented.

## Acceptance Criteria

- [ ]  **ResourceTracker.svelte** component created and:
    - Accepts `character` prop (full record with `resources` JSON field)
    - Displays all resources found in `character.resources` object
    - No hardcoded resource list; iterates over whatever is in the JSON
    - Renders gracefully if `resources` is null or empty
- [ ]  **Resource display format** for each resource:
    - Shows resource name (e.g., "Hit Dice", "Action Surge", "Spell Slots (1st Level)")
    - Shows current / max value (e.g., "4 / 4")
    - For Hit Dice, also shows die type (e.g., "d10" → "4 / 4 (d10)")
    - For Spell Slots, groups by spell level and displays each level separately
    - Uses visual indicators (circles, bars, or boxes) for remaining uses (optional but recommended)
- [ ]  **Visual design**:
    - Responsive layout: stacks vertically on mobile, may use columns on tablet+
    - Clear spacing and visual hierarchy
    - Color coding optional (e.g., green for full, yellow for partial, red for depleted)
    - Accessible semantic HTML
- [ ]  **Spell Slots organization** (if present in resources):
    - Spell slots grouped under a "Spell Slots" section or expandable list
    - Each slot level (1st through 9th) displayed as separate row
    - Sorted by spell level (ascending)
    - Format: "1st Level: 4 / 4", "2nd Level: 3 / 3", etc.
    - Only display spell levels with non-zero max
- [ ]  **Exported props**:
    - `character` (required): full character record
    - `readonly` (optional, default true): if true, no increment/decrement buttons; if false, shows buttons (for future mutations)
- [ ]  **Handling edge cases**:
    - If a resource has `current > max`, display as-is with visual warning or different color
    - If a resource is depleted (`current === 0`), still display with visual indicator (e.g., grayed out)
    - If `resources` field is missing or null, render "No resources tracked" message
- [ ]  **Storybook stories** created:
    - Default story with Sir Nabo fixture (full resources)
    - Story with mixed resources (some depleted, some partial)
    - Story with empty resources
    - Story with only spell slots
    - All stories render cleanly at 375px, 768px, 1920px viewports

### Resource Name Formatting

Create a utility function to convert snake_case resource keys to display names:

```jsx
// src/lib/utils/resource-format.js
export function formatResourceName(key: string): string {
  return key
    .replace(/_/g, ' ')
    .replace(/\b\w/g, (c) => c.toUpperCase());
    // "hit_dice" → "Hit Dice"
    // "second_wind" → "Second Wind"
}

export function formatSpellLevel(level: string): string {
  const levelNum = parseInt(level.replace('level_', ''));
  const ordinals = ['', '1st', '2nd', '3rd', '4th', '5th', '6th', '7th', '8th', '9th'];
  return `${ordinals[levelNum] || levelNum} Level`;
}
```

### Spell Slots Sorting

Sort spell slots by level number:

```jsx
$: sortedSpellSlots = Object.entries(spellSlots)
  .sort(([keyA], [keyB]) => {
    const levelA = parseInt(keyA.replace('level_', ''));
    const levelB = parseInt(keyB.replace('level_', ''));
    return levelA - levelB;
  });
```

### Edge Cases

- **current > max**: Display as-is; add visual warning (e.g., orange text). This can happen if spellcasting is temporarily boosted.
- **Negative current**: Unlikely but gracefully handle (display as-is, may indicate debt or error state).
- **Missing `die` field**: Spell slots and some features don't have a die; omit this field from display.
- **Nested spell slots**: Spell Slots are nested under `spell_slots` key; special handling to flatten and display correctly.
