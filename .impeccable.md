## Design Context

### Users
- **Stage Operators**: Need high-efficiency, low-latency control surfaces for live production.
- **Cast (Players/DMs)**: Need mobile-first, glanceable game state that doesn't distract from the roleplay.
- **Audience**: Need high-clarity, visually exciting overlays that communicate state at a distance.

### Brand Personality
- **Immersive**: Everything feels like it belongs in a high-stakes fantasy world.
- **Precise**: Information (HP, Dice) is accurate and updates in real-time.
- **Electric**: High-contrast colors and springy motion make the interface feel "alive."

### Aesthetic Direction
- **Theme**: "Digital Grimoire" — Dark Mode by default (`--black`).
- **Typography**: High-impact headings (`Bebas Neue`) paired with high-readability UI text. Monospaced numbers (`JetBrains Mono`) for all game data.
- **Accents**: Functional color coding (Red for Damage, Cyan for Healing, Purple for Magic/Dice).

### Design Principles
1. **Performance First**: Zero perceived latency between a Stage click and an Audience update.
2. **High-Contrast Clarity**: Text and stats must be legible even on compressed 720p streams.
3. **Tactile Motion**: Use springy, physical transitions (`--t-spring`) to make digital stats feel like physical objects.
4. **Surface-Specific Density**: 
   - *Stage*: High density, compact controls.
   - *Cast*: Medium density, finger-friendly targets.
   - *Audience*: Low density, high visual impact.
