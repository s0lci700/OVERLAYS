## Design Context

### Users
- **Stage Operators**: Need high-efficiency, low-latency control surfaces for live production.
- **Cast (Players/DMs)**: Need mobile-first, glanceable game state that doesn't distract from the roleplay.
- **Audience**: Need high-clarity, visually exciting overlays that communicate state at a distance.

### Brand Personality
- **Immersive**: Everything feels like it belongs in a high-stakes fantasy world.
- **Precise**: Information (HP, Dice) is accurate and updates in real-time.
- **Electric**: High-contrast colors and springy motion make the interface feel "alive."

### Aesthetic Direction
- **Theme**: "Digital Grimoire" — Dark Mode by default (`--black`).
- **Typography**: High-impact headings (`Bebas Neue`) paired with high-readability UI text. Monospaced numbers (`JetBrains Mono`) for all game data.
- **Accents**: Functional color coding (Red for Damage, Cyan for Healing, Purple for Magic/Dice).

### Design Principles
1. **Performance First**: Zero perceived latency between a Stage click and an Audience update.
2. **High-Contrast Clarity**: Text and stats must be legible even on compressed 720p streams.
3. **Tactile Motion**: Use springy, physical transitions (`--t-spring`) to make digital stats feel like physical objects.
4. **Surface-Specific Density**:
   - *Stage*: High density, compact controls.
   - *Cast*: Medium density, finger-friendly targets.
   - *Audience*: Low density, high visual impact.

## Cast Player Layer — "The Living Archive"

This section is the canonical design reference for `(cast)/players/[id]` — all 4 tabs (Home, Skills, Magic, Notes) as a unified surface. When building or reviewing any Cast Player component, these rules are the source of truth.

### Concept
The Living Archive is a **silent custodian** — a precise, authoritative mobile surface that "pulses" with life only when game state changes. Players glance at it mid-roleplay; it must never demand attention.

### Color Palette
The Cast layer uses a two-tier accent system: **amber for all structural chrome**, **cyan for live-state data only**.

| Role | Value | MD3 token | Use |
|------|-------|-----------|-----|
| Background | `#0d0d15` | `surface-container-lowest` | Page bg, nav bg |
| Panel low | `#1b1b23` | `surface-container-low` | Content cards, sections |
| Panel mid | `#1f1f27` | `surface-container` | Elevated panels |
| Panel high | `#292932` | `surface-container-high` | Stat strips, expanded states |
| Primary text | `#f0f0f0` | `tertiary` | All primary content |
| Secondary text | `#bac9cc` | `on-surface-variant` | Labels, subtitles |
| **Amber** | **`#C8944A`** | `primary-container` | Nav active, section dividers, left rails, proficiency, filter chips, all structural chrome |
| Amber pale | `#E8B060` | `primary` | Ability score modifiers, data readouts |
| Amber glow | `rgba(200,148,74,0.4)` | — | Spell slot breathing shadows |
| **Cyan** | **`#00E5FF`** | — | **Live-state ONLY**: HP/AC display + `update-pulse` animation |
| Lavender | `#cdbdff` | `secondary` | Magic-specific elements (mana gem glow) |
| Deep glass | `oklch(12% 0.01 240)` | `brand-cyan` | `.trace-cyan` — etched glass panel fill (near-black) |

> **Luminous economy**: Cyan communicates "this value is changing right now." Amber communicates "this is part of the interface." Never use cyan decoratively. Never use amber on live HP/AC.

> **Tailwind config**: All 4 files must have `"primary-container": "#C8944A"` and `"primary": "#E8B060"` in their tailwind config overrides. HP/AC displays must use hardcoded `text-[#00E5FF]`, not semantic `text-primary`.

### Typography

| Role | Font | Weight | Treatment | Use |
|------|------|--------|-----------|-----|
| Character names, section titles | `Noto Serif` | 600 | `tracking-[-0.02em]` | Identity layer — fantasy gravitas |
| Labels, nav, caps UI | `Space Grotesk` | 500–700 | All-caps, `tracking-[0.1em]` to `tracking-[0.2em]` | All UI chrome |
| Tactical data (HP, AC, modifiers) | `JetBrains Mono` | 700 | `tracking-[0.05em]` | Numbers that must be read at a glance |
| Spell/skill prose descriptions | `Inter` | 400 | `leading-relaxed` | Longer readable text |

> `Noto Serif` is **Cast-layer only**. Stage and Audience continue to use `Black Ops One` / `--font-display`.

### Geometry — Zero Radius
- `border-radius: 0px` everywhere. No rounded cards, no rounded inputs.
- Only exception: `border-radius: 9999px` (`full`) for pills and avatar clips.
- Hexagonal portrait frame: `clip-path: polygon(2% 0%, 98% 0%, 100% 50%, 98% 100%, 2% 100%, 0% 50%)`

### Surface Language — Deep Glass
All panels use the same glass treatment for visual coherence:
```
background: #1b1b23 at 60% opacity
backdrop-filter: blur(40px)
border: 1px solid rgba(255,255,255,0.05) — rgba(255,255,255,0.1)
```
Headers and nav use `#0d0d15` at 60% opacity + 40px blur (deeper, not frosted).

### Luminous Economy — When to Glow
The rule is **zero glow at idle** — glows are reserved to communicate live state changes:

| State | Effect |
|-------|--------|
| **Idle** | No glow. Data is crisp white on deep glass. |
| **Live data** (HP/AC updating) | `update-pulse`: shimmer border + `box-shadow 0 0 10px rgba(0,229,255,0.4)` |
| **Active spell slot** | `breathing-glow`: 3s ease-in-out scale 1→1.02 + opacity 0.9→1 |
| **HP critical (<25%)** | Slow 2s breathing animation on HP fill |
| **Status pill** | `glass-shimmer`: left-to-right shimmer sweep every 3s |
| **Ambient layer** | HUD scanlines at 3% opacity on Magic tab (optional per-screen) |

### Proficiency Indicators — Shape-Based
Proficiency is communicated through **shape + color**, never color alone:

| State | Indicator |
|-------|-----------|
| Non-proficient | `triangle-outline` — hollow CSS triangle in white/20 |
| Proficient | `triangle-marker` — solid amber `#C8944A` triangle |
| Expertise | Two solid amber triangles side by side |

### Spell Slot Shapes — Accessible by Shape
Spell slots use geometric clip-paths so depleted/available is readable without color:

| Level | Shape | Clip-path |
|-------|-------|-----------|
| L1 | Triangle | `polygon(50% 0%, 0% 100%, 100% 100%)` |
| L2 | Diamond | `polygon(50% 0%, 100% 50%, 50% 100%, 0% 50%)` |
| L3 | Hexagon | `polygon(25% 0%, 75% 0%, 100% 50%, 75% 100%, 25% 100%, 0% 50%)` |

Filled = glowing cyan with `drop-shadow(0 0 8px rgba(0,229,255,0.4))`. Depleted = hollow `border border-primary-container/40`.

### Active State — Left Rail Priority
Content priority is communicated through left-border weight:
- **Priority / active**: `border-l-2 border-[#C8944A]` — solid amber
- **Secondary**: `border-l-2 border-[#C8944A]/30` — dim amber
- **Inactive / used**: `border-l-2 border-transparent` — no rail

### Navigation — Bottom Bar (4 Tabs)
Fixed bottom bar: HOME / SKILLS / MAGIC / NOTES

```
Inactive: text-[#f0f0f0]/60, outline icon variant
Active:   text-[#C8944A], border-t-2 border-[#C8944A],
          shadow-[0_-8px_15px_-5px_rgba(200,148,74,0.25)],
          bg-[#C8944A]/5
```

All tabs: `active:scale-95`, `transition-all`, `duration-200`.

### Persistent Header
All 4 screens share the same fixed header structure:
- Left: Character name in `JetBrains Mono Bold` uppercase + class/level subtitle in `Space Grotesk`
- Right: `HP X/Y | AC N` in `JetBrains Mono Bold` cyan
- Height: `h-16` (64px), deep glass treatment

### Scrollbar Style
```css
::-webkit-scrollbar { width: 4px; }
::-webkit-scrollbar-track { background: transparent; }
::-webkit-scrollbar-thumb { background: #C8944A44; }
```
Hide scrollbar entirely on filter chips: `no-scrollbar` pattern.

### Content Patterns per Tab

**Home** — Tactical overview: stat strip + ability score grid + spell slots + signature skills
**Skills** — Dense ledger: 56px rows, value + name + ability + proficiency triangle(s)
**Magic** — Spell slots header → spellbook by level → expandable spell detail card with numeric data grid
**Notes** — Quest log + session chronicle (date-stamped) + scratchpad (JetBrains Mono textarea)

### Coherence Rules
When building any Cast Player component, enforce:
1. **0px radius** — no softness anywhere except avatar hex and full pills
2. **Deep glass panels** — always `backdrop-blur-[12px]`, near-opaque `rgba(13,13,21,0.6)` backgrounds
3. **Noto Serif for identity** — character names and section titles only
4. **Amber chrome, cyan live** — `#C8944A` for all structural UI; `#00E5FF` reserved for HP/AC display only
5. **Luminous economy** — only glow on live state change, never decoratively
6. **Shape-based accessibility** — proficiency and spell slots readable without color
